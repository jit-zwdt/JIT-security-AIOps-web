Weblogic jdbc自动发现jdbc监控项

  1. Weblogic配置
    对于windows版本, 编辑WL_DOMAIN_HOME/bin/setDomainEnv.cmd，在文件结尾加入下面几行：(注：WL_DOMAIN_HOME为应用程序域安装的位置) 
    set JAVA_OPTIONS=%JAVA_OPTIONS% -Djava.rmi.server.hostname=*.*.*.*        #Weblogic主机IP
set JAVA_OPTIONS=%JAVA_OPTIONS% -Djavax.management.builder.initial=weblogic.management.jmx.mbeanserver.WLSMBeanServerBuilder
set JAVA_OPTIONS=%JAVA_OPTIONS% -Dcom.sun.management.jmxremote=true
set JAVA_OPTIONS=%JAVA_OPTIONS% -Dcom.sun.management.jmxremote.port=12345        #JMX_PORT
set JAVA_OPTIONS=%JAVA_OPTIONS% -Dcom.sun.management.jmxremote.ssl=false
set JAVA_OPTIONS=%JAVA_OPTIONS% -Dcom.sun.management.jmxremote.authenticate=false
    注：hostname=TWeblogic服务器IP地址，该IP必须能与ZABBIX Server能通信，某些情况下IP经过映射转换，那么需要使用转换后的地址。如果同一台机器上有运行多个Weblogic程序，每个Weblogic的监控操作步骤都一样，但是这里的监听端口JMX_PORT每个Weblogic程序必须不同，否则端口会冲突
    对于Linux版本, 编辑WL_DOMAIN_HOME/bin/setDomainEnv.sh(注：WL_DOMAIN_HOME为应用程序域安装的位置) 在文件结尾加入下面几行：
    JAVA_OPTIONS="${JAVA_OPTIONS} -Djavax.management.builder.initial=weblogic.management.jmx.mbeanserver.WLSMBeanServerBuilder"
JAVA_OPTIONS="${JAVA_OPTIONS} -Dcom.sun.management.jmxremote=true"
JAVA_OPTIONS="${JAVA_OPTIONS} -Dcom.sun.management.jmxremote.port=12345"        #JMX端口号
JAVA_OPTIONS="${JAVA_OPTIONS} -Dcom.sun.management.jmxremote.ssl=false"
JAVA_OPTIONS="${JAVA_OPTIONS} -Dcom.sun.management.jmxremote.authenticate=false"
export JAVA_OPTIONS
    注：hostname=TWeblogic服务器IP地址，该IP必须能与ZABBIX Server能通信，某些情况下IP经过映射转换，那么需要使用转换后的地址。如果同一台机器上有运行多个Weblogic程序，每个Weblogic的监控操作步骤都一样，但是这里的监听端口JMX_PORT每个Weblogic程序必须不同，否则端口会冲突
    如果你的应用不是布署在默认的应用服务器上，而是新增了一个应用服务器布署的话，应该使用下面这个方法: 进入weblogic控制台->环境->服务器->”你新增的服务器”->配置->服务器启动。在“参数”的输入框内输入：
    -Dcom.sun.management.jmxremote.port=JMX_PORT
-Djava.rmi.server.hostname=JMX_HOST
-Dcom.sun.management.jmxremote.authenticate=false  
-Dcom.sun.management.jmxremote.ssl=false
-Djavax.management.builder.initial=weblogic.management.jmx.mbeanserver.WLSMBeanServerBuilder

    然后保存，激活更改，再重启这个应用服务器就开启JMX了。
